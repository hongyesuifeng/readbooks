# 深度阅读分析框架

> 基于 Adier《如何阅读一本书》分析阅读+主题阅读框架，融合批判性思维与跨领域映射。
> 适用于所有非虚构类书籍的深度分析，尤其针对游戏设计、软件工程等实践类书籍。

---

## 一、阅读层次标准（五层模型）

对应项目规则中定义的笔记深度五层标准：

| 层 | 名称 | 特征动词 | 判断标准 |
|----|------|---------|---------|
| 1 | 复述 | "书上说..." | 能说出书中有什么内容 |
| 2 | 解释 | "这意味着..." | 能用自己的话说明设计意图 |
| 3 | 取舍 | "作者选择了X而放弃了Y，因为Z" | 识别约束条件、揭示代价 |
| 4 | 洞察 | "这揭示了一个普遍规律: 当A时，B比C更好，因为D" | 从具体案例提炼出可迁移的原理 |
| 5 | 改变认知 | "读完这篇笔记，我对X的理解从A变成了B" | 产出可指导行动的认知升级 |

**每本书的分析必须包含至少3个第4层洞察和1个第5层认知改变。**

## 二、深度分析的七个维度

每本书的分析必须覆盖以下七个维度。无法覆盖的维度要明确说明原因。

### D1: 世界观提取

**目标**：找出作者不需要论证就默认成立的底层假设。

**操作**：
- 这本书把"好游戏"定义为什么？这个定义是作者自己说的还是隐含的？
- 作者的职业经历（3A vs 独立、日本 vs 西方、程序员 vs 设计师）如何塑造了他的视野盲区？
- 如果换一个背景完全不同的人来写这个主题，会有什么不同？

**输出格式**：
```
**作者的底层世界观**：[一句话]
**基于此推导出的必然结论**：[逻辑链]
**这个世界的反面**：[什么情况下不成立]
```

### D2: 概念发生学

**目标**：追溯核心概念的来源和演进，而不是把它当作"本来就有"的。

**操作**：
- 这个概念最早由谁在什么书中提出？当时在解决什么问题？
- 概念提出后经历了什么修改？（谁反驳了它？谁扩展了它？）
- 概念在游戏行业中的实际接受度如何？（所有人都用？还是仍有争议？）

**输出格式**：
```
**概念**：[名称]
**来源**：[作者/书/时间] - [原本解决的问题]
**演进**：[关键节点1 → 关键节点2 → 当前状态]
**当前争议**：[如果有]
```

### D3: 书籍间对话

**目标**：不止列出"相关的书"，而是构建真实的对话。

**操作**：
- 如果A书的作者和B书的作者在同一个房间，他们会在哪里同意，在哪里吵起来？
- A书说X是对的，B书中有没有明说或暗示X是不对的？
- 同一个概念在两本书中用了不同的术语——它们是否在说同一件事但表述不同？还是一个更深刻？

**输出格式**：
```
**对话对**：《A书》vs《B书》
**共识点**：[在哪里它们说了一样的话]
**分歧点**：[在哪里它们的答案相反] - [分歧的根源是什么？背景差异？问题域不同？]
**谁的答案更好**：[在什么条件下]
```

### D4: 反例检验

**目标**：用真实游戏案例检验理论的适用边界。

**操作**：
- 有什么成功游戏明显违反了这个原则？
- 反例说明了什么？（是原则有漏洞、有额外前提条件、还是反例其实是更深层原则的体现？）
- 有没有"遵守了这个原则却失败的"案例？

**输出格式**：
```
**声称的原则**：[来自本书]
**反例游戏**：[游戏名]
**违反了什么**：[具体违反的部分]
**为什么反例能成功**：[更深层的解释]
**对原则的修正**：[加了什么限定条件后原则依然成立]
```

### D5: 跨领域映射

**目标**：把游戏设计概念映射到非游戏领域，验证其普适性。

**操作**：
- 这个概念在软件架构/产品设计/管理/写作中有没有对应的概念？
- 映射到其他领域后，概念变得更清晰还是更模糊？
- 跨领域映射揭示了游戏领域的什么独特性？

**输出格式**：
```
**游戏概念** → **领域B概念**
**映射准确度**：[高/中/低] - [因为...]
**相互启示**：[从B领域回头看游戏设计，有什么新认识]
```

### D6: 实践转化

**目标**：从抽象到具体——不光说"可以借鉴"，而是指定到文件/类/函数/参数。

**操作**：
- 这个概念在我的哪个具体项目/模块/文件中能落地？
- 如果现在立即实现，需要改哪几行？参数设为什么？
- 如何验证这个借鉴是否有效？（KPI？玩家反馈指标？定性感受？）

**输出格式**：
```
**借鉴目标**：[项目/模块/文件]
**具体改动**：[改什么] - [怎么改] - [期望效果]
**验证方式**：[如何知道改动成功了]
```

### D7: 元认知记录

**目标**：记录阅读过程中自己认知的变化，这对"第5层"至关重要。

**操作**：
- 读之前我以为是什么？读之后发现真相是什么？
- 哪个句子/段落让我产生了"原来如此！"的顿悟？
- 我对这本书的评价在阅读过程中有没有发生变化？

**输出格式**：
```
**读前认知**：[我之前以为...]
**读后认知**：[现在我认为...]
**关键转折点**：[哪个观点触发了变化]
**残留疑问**：[还有什么没想通的]
```

## 三、主题阅读维度（多书综合）

当分析多本同主题书籍时，必须进行主题阅读层分析：

### T1: 核心问题矩阵

针对每本书，回答同一个问题："如果只能从这本书带走一句话，那句话是什么？"

| 书 | 一句话 | 隐含的世界观 |
|----|--------|-------------|
| ... | ... | ... |

### T2: 术语翻译表

同一概念在不同书中的不同名称，建立等价或层次关系：

| 概念本质 | 书A的叫法 | 书B的叫法 | 关系（等价/A包含B/A更深刻） |
|---------|----------|----------|--------------------------|
| ... | ... | ... | ... |

### T3: 框架层级结构

不是并列"这14本书各自说了什么"，而是：
- 哪些是**元框架**（提供分析其他框架的框架）？
- 哪些是**操作框架**（教你一步一步做什么）？
- 哪些是**专项深入**（只深入一个维度）？

### T4: 辩论矩阵

找出真正的分歧点——在对同一个问题的回答上，哪些书站在对立面？

| 核心问题 | 支持的立场和书 | 反对的立场和书 | 你的判断 |
|---------|--------------|--------------|---------|
| ... | ... | ... | ... |

## 四、执行流程

### 笔记生成标准结构（v3.0 两大部分强制结构）

```
# Part 1：书籍内容梳理归纳
  ├── 📖 基本信息
  ├── 📝 内容概要（书籍定位、核心主题）
  ├── 🧠 知识架构（Mermaid mindmap）
  ├── ✍️ 分章笔记（每章独立一节，不可合并/跳过）
  │    格式：核心问题→关键概念→原理→设计取舍→章间关联
  └── 章节速查表

# Part 2：深度分析与思考
  ├── D1-D7 七个维度全部覆盖
  └── 📊 学习总结（第5层认知改变 + ≥3个第4层洞见）
```

**关键原则**：Part 1 先客观呈现"书上说了什么"（不跳章、不合并），Part 2 再做"这意味着什么，对不对，怎么用"。**两部分不可互相替代，不可跳过任何一章。**

### 单本书深度分析流程

```
阶段1: 世界观提取（5分钟）
  → 回答D1：作者把什么当作理所当然？

阶段2: 关键概念拆解（15分钟）
  → 选取3-5个核心概念
  → 回答D2：每个概念的来源和演进

阶段3: 作者与作者的对话（10分钟）
  → 选取2-3本已分析的书
  → 回答D3：在哪里同意，在哪里吵

阶段4: 反例检验（10分钟）
  → 回答D4：什么游戏违反了这个原则却成功了

阶段5: 跨领域映射（10分钟）
  → 回答D5：这个游戏设计概念在软件/管理中叫什么

阶段6: 实践转化（10分钟）
  → 回答D6：下个Sprint能改哪一行

阶段7: 元认知记录（5分钟）
  → 回答D7：读完我什么变了

总时间：约1小时/本（不含信息搜索时间）
```

### 多书主题分析流程

```
阶段1: 核心问题矩阵 → 每本书的一句话提炼
阶段2: 术语翻译表 → 统一概念名称
阶段3: 框架层级结构 → 建立元-操作-专项层次
阶段4: 辩论矩阵 → 识别真正的分歧点
阶段5: 综合判断 → 哪些认知被多本书共同确认，哪些仍在争议
```

## 五、禁止写法重申

以下写法出现即不合格，必须重写：

- ~~"这本书让我收获很多"~~ → 必须说出：收获了**什么**，**改变了哪个认知**
- ~~"作者的观点很有启发性"~~ → 必须说出：启发了**什么**，会导致**什么不同的行动**
- ~~"与《X》有关联"~~ → 必须说出：在**哪里**同意，在**哪里**分歧
- ~~"可以借鉴到项目中"~~ → 必须指定到**具体文件/类/函数/参数**
- ~~每章笔记长度均等~~ → 重要的章节应该比不重要的长3-5倍
- ~~"X设计得非常好"~~ → 必须说出：好在**什么条件**下，对**谁**好，**代价**是什么
