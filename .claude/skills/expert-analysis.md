# 专家分析生成

自动为书籍生成多专家视角深度分析，整合不同专业领域的见解。

## 参数

- `bookName`: 书名（必需）
- `bookType`: 书籍类型（可选，自动检测）
  - 可选值：`tech`（技术类）、`theory`（理论类）、`practice`（实践类）、`business`（商业类）、`game`（游戏类）
- `expertTypes`: 专家类型（可选，默认自动选择）
  - 可选值：`cs`（计算机科学）、`ai`（人工智能）、`business`（商业科技）、`game_client`（游戏客户端）、`game_industry`（游戏行业）

## 示例

```
专家分析 设计模式
专家分析 "人月神话" tech
专家分析 "深度学习" ai --experts cs,ai
```

## 输出格式

生成的专家分析包含以下部分：

### 1. 专家选择说明
说明为什么选择这些专家来分析这本书

### 2. 各专家深度分析
每位专家的分析包含：
- **核心洞察**：3-5个关键洞察点
- **深度分析**：详细的分析维度和观点
- **独特视角**：该专家提供的独特见解

### 3. 专家观点整合
- **共识点**：多个专家共同认可的观点
- **分歧点**：专家们观点不同的地方及原因
- **综合结论**：整合各专家观点的总结

## 执行步骤

1. **分析书籍类型**：根据书名和内容判断书籍类型
2. **选择相关专家**：根据类型选择主专家和辅助专家
3. **生成专家分析**：为每位专家生成深度分析
4. **整合专家观点**：识别共识和分歧，生成综合结论
5. **格式化输出**：按照标准格式输出分析结果

## 质量标准

- ✅ 专家选择与书籍主题高度相关
- ✅ 每位专家提供独特且有价值的见解
- ✅ 分析深度适中，既专业又易懂
- ✅ 观点整合清晰，有明确的共识和分歧
- ✅ 提供实践指导和行动建议

## 使用场景

1. **新书阅读前**：了解专家对书籍的评价和期待
2. **阅读过程中**：从不同专家角度理解书中的观点
3. **阅读完成后**：整合专家观点，形成自己的认识
4. **笔记整理**：作为读书笔记的专家视角部分
