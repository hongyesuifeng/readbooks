## T1: 每本书一句话 + T3: 四层知识结构

| 层 | 书 | 一句话 |
|----|-----|--------|
| **认知机制层** | **思考快与慢** | 你有两个大脑：系统1(快/直觉/容易错)和系统2(慢/理性/懒)——大多数决策是系统1做的不经审视的 |
| | **第一性原理** | 把问题拆到不能再拆的基本真理，从那里出发重建——不要接受"大家都是这么做的"作为论证 |
| **概率与不确定性层** | **黑天鹅** | 极端罕见事件比你的所有模型预测的都更常见、影响更大——而人类擅长在事后骗自己说"其实是可以预见的" |
| | **灰犀牛** | 与黑天鹅相反——大概率、高影响的威胁就在你面前，但人类集体选择视而不见（如气候变化、技术债务） |
| **信息与真相层** | **后真相** | 当"感觉真实"比"事实真实"更能驱动行为时，社会如何运作 |
| | **真相** | 人们对同一个事实可以得出完全相反的结论——不是因为他们看到了不同的事实，而是因为他们用了不同的**解释框架** |
| **实践与关系层** | **真本事** | 从"会工作"到"会赚钱"的关键转变是什么 |
| | **真需求** | 用户说的"需求"和真正驱动他们行为的"需求"之间的差距 |
| | **非暴力沟通** | 冲突的根源不是"我们的需求矛盾"而是"我们表达需求的方式制造了本不存在的矛盾" |
| | **幸福的婚姻** | 婚姻中决定成败的不是冲突有多少，而是**冲突后的修复能力** |
| | **爱的博弈** | 亲密关系可以用博弈论框架来分析——信任=反复博弈中持续合作策略的累积 |

## T2: 核心辩论 — 11本书共同指向的张力

### 辩论: 认知偏见是"bug"还是"feature"？

| 立场 | 代表书 | 论点 |
|------|--------|------|
| **偏见=bug** | 思考快与慢 | 系统1的启发式判断导致可预测的系统性错误——锚定效应、可得性偏差、确认偏误——这些是"认知的软件bug"，需要系统2修复 |
| **偏见=feature** | 黑天鹅(隐含) + 真相(隐含) | 系统1之所以存在，因为它能在"远古草原上听到草丛沙沙声就立刻跑"的场景中救你的命。在那种情境下，"宁可误判一千次也不能漏判一次"是最优策略 |
| **你的判断** | — | 偏见的性质取决于环境。在统计概率环境中（投资、医学诊断），偏见=bug。在社会生存环境中（声誉、信任、亲密关系），偏见=feature。优秀的思维者不是"没有偏见"，而是**知道哪种环境在调用哪种偏见** |

## D1/D3/D5/D7: 关键书深度维度

### 《思考快与慢》— Kahneman

**D1: 世界观**：Kahneman把"认知"看作一个**由两个独立代理组成的裁决系统**——系统1负责日常运营（高效但犯系统性错误），系统2负责监督（理性但懒惰）。你的大多数"理性决策"是系统1做的，系统2只是在事后编一个"我当然是慎重考虑过才这么做的"的叙事。

**D3: 对话 — 思考快与慢 vs Koster《A Theory of Fun》**

| 概念 | Kahneman | Koster | 关系 |
|------|----------|--------|------|
| 自动化的认知 | 系统1 | Grok（深度内化） | Koster说的"Grok"就是"把一个系统2的任务练到变成系统1的自动反应"——这就是一切游戏技能的认知基础 |
| 为什么游戏有趣 | (未直接涉及) | 大脑在识别新模式 | 从Kahneman的框架看Koster：游戏有趣的本质=给系统1不断提供"可识别的模式"（不用唤醒系统2），但又偶尔要求系统2介入（策略/解谜）——**游戏的节奏本质上是系统1和系统2的交替活跃** |

**D5: 跨领域映射**：Swink的ADSR模型描述的"游戏操控感"本质上就是系统1的反应——当Attack=2帧时，玩家的系统1在2帧内完成了"按下按键→预期结果→确认结果→准备下一个输入"的闭环。如果要求系统2参与（超过约300ms），手感就"重"了。**游戏手感参数化的神经科学基础：Attack<认知系统1阈值。**

### 《第一性原理》

**D4: 反例**：马斯克用第一性原理分析"火箭为什么这么贵"得出了SpaceX。但无数创业者用第一性原理分析出"我的创业想法一定成立"——然后失败了。不是第一性原理这个工具有问题，而是**大多数人用第一性原理时停在了一个"足够支持我的信念"的基本真理，而非真正不可再拆的基本真理。** 第一性原理的敌人不是复杂性，而是**"停在假的第一性"。**

**D3: 对话 — 第一性原理 vs 设计模式**
"第一性原理"是重新从底层真理推导——设计模式是"别从底层推导了，这是前人已经验证过的解决方案"。两者的张力代表了一切创造性工作中的永恒选择题：**什么时候你应该自己重新思考？什么时候你应该信任前人的解决方案？** 答案是：当"前人的解决方案"的上下文与你的上下文匹配度高→用模式。当匹配度低→用第一性原理重新推导。

### 《黑天鹅》— Taleb

**D7: 元认知**：
**读前**："黑天鹅=罕见但影响大的意外事件。"
**读后**："黑天鹅的关键不是'它会发生'——而是'发生之后所有人都会说其实它是可以预见的'——而这是后验的理性的幻觉。Taleb教会我的不是预测黑天鹅，而是**建立反脆弱系统**——不是预测冲击，而是被冲击后变得更强。"

### 非暴力沟通 / 幸福的婚姻 / 爱的博弈 — 关系三件套

**T5: 关键发现**：这三本书讨论的"沟通/修复/信任"机制，在游戏设计的多人模式中完全同构。
- NVC的"观察→感受→需求→请求" = 多人游戏中"我看到队友在打野→我感到失望→因为我需要前线有人→请求你去前线"
- 幸福的婚姻的"修复尝试" = 《守望先锋》中的"没关系，下一波打好"——游戏中的社交修复机制决定了团队的粘性
- 爱的博弈的"信任=持续合作博弈的累积" = 任何多人合作游戏的底层社交模型
