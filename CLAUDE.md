# CLAUDE.md

此文件为 Claude Code (claude.ai/code) 在此代码仓库中工作时提供指导。

## 项目概述

这是"ReadBooks"项目 - 一个个人读书笔记与学习记录平台。用于系统性地记录阅读过程中的思考、总结和知识体系构建。

## 项目目标

1. **读书笔记管理** - 为每本书创建详细的读书笔记
2. **知识架构生成** - 使用 Mermaid 等工具绘制知识体系图
3. **深度衍生思考** - 针对书籍内容进行延伸思考和多角度分析
4. **知识关联网络** - 建立书籍间的关联关系，形成知识图谱
5. **后续阅读规划** - 基于当前书籍推荐后续阅读和分析方向
6. **学习进度跟踪** - 记录阅读状态和个人成长轨迹

## 工作流程

当用户要求为某本书生成读书笔记时：

1. **信息收集** - 使用 WebSearch 搜索书籍相关信息
2. **模板选择** - 根据书籍类型选择合适的模板
3. **专家分析** - 自动选择相关专家并生成深度分析
4. **知识关联** - 分析与已读书籍的关联关系
5. **后续推荐** - 基于书籍内容推荐后续阅读路径
6. **笔记生成** - 创建符合模板的完整读书笔记
7. **质量检查** - 使用质量标准检查清单验证笔记质量
8. **文件组织** - 将笔记保存到 `books/` 目录下
9. **知识图谱更新** - 更新全局知识关联图谱

详细流程见 `.claude/workflows/读书笔记标准流程.md`

## 读书笔记标准格式

每本书的笔记必须包含：

### 基础结构（必需）
- 📚 基础信息（书名、作者、出版社等）
- 📖 内容概要（书籍简介、核心主题、主要章节）
- 🧠 知识架构（Mermaid 思维导图）
- ✍️ 读书笔记（重点摘录、个人思考、实践应用）
- 📊 学习总结（收获、观念改变、未来行动）

### 深度分析（⭐⭐⭐⭐以上）
- 💭 深度衍生思考（核心观点延伸、多角度分析、创新思考）
- 🎓 专家视角分析（多专家深度分析和观点整合）
- 🔗 知识关联网络（与已读书籍的关联、概念映射）
- 📚 后续阅读路径规划（直接延伸、交叉验证、实践补充）

### 质量标准
- 所有笔记应达到⭐⭐⭐⭐以上质量等级
- 优秀笔记应达到⭐⭐⭐⭐⭐质量等级
- 质量标准详见 `templates/笔记质量标准.md`

## 常用命令

### 读书笔记生成
- **创建新书笔记**: `读书笔记 {书名}` - 自动搜索并生成完整笔记
- **专家分析**: `专家分析 {书名}` - 生成多专家视角深度分析
- **快速笔记**: `读书笔记 {书名} simple` - 生成简化版笔记

### 知识管理
- **查看知识图谱**: 查看 `books/知识关联图谱.md`
- **质量检查**: 使用 `templates/笔记质量标准.md` 检查笔记质量
- **更新关联**: 阅读新书籍后更新知识关联图谱

### 工作流程
- **标准流程**: 参考 `.claude/workflows/读书笔记标准流程.md`
- **模板选择**: 根据书籍类型选择合适的模板
- **专家选择**: 根据书籍主题自动选择相关专家

## 注意事项

1. **始终使用中文** - 所有对话和笔记内容都使用中文
2. **模板化生成** - 使用标准化模板确保笔记格式一致
3. **个性化内容** - 在标准格式基础上添加个人思考和感悟
4. **知识可视化** - 重视思维导图和架构图的绘制
5. **版权尊重** - 摘录内容要注明出处，避免大段引用

## 专家Agent系统

项目内置了多位资深专家Agent，可从不同专业视角分析书籍内容，帮助提炼核心观点。

### 可用专家

| 专家 | 领域 | 专长 |
|------|------|------|
| 张明远教授 | 计算机科学 | 系统架构、算法、编程语言理论 |
| 李思源博士 | 人工智能 | 机器学习、深度学习、AI应用 |
| 王建华教授 | 商业科技 | 科技战略、商业模式、投资分析 |
| 陈晓峰 | 游戏客户端 | 游戏引擎、图形渲染、性能优化 |
| 周文博 | 游戏行业 | 游戏设计、市场分析、产业发展 |

详细定义见 `agents/experts.md` 和 `agents/system.md`

### 使用方式

1. **单专家分析**:
   ```
   请用张明远教授的视角分析《编译原理》
   ```

2. **多专家分析**:
   ```
   请从计算机科学和人工智能角度分析这本书
   ```

3. **专家圆桌讨论**:
   ```
   请组织专家圆桌讨论这本书的核心价值
   ```

### 在笔记中添加专家视角

使用 `templates/专家分析框架.md` 中定义的标准格式：

```markdown
## 🎓 专家视角深度分析

### 张明远教授（计算机科学）
#### 核心洞察
- 洞察1
- 洞察2

#### 深度分析
**专家观点**：详细分析
**理论支撑**：相关理论
**实践案例**：具体案例

#### 独特视角
独特见解

### 李思源博士（人工智能）
[类似结构]

## 💡 专家观点整合
### 共识点
- 共识1
- 共识2

### 分歧点
- 分歧1及原因

### 综合结论
整合观点
```

根据书籍类型自动选择：
- **计算机科学类** → 张明远教授（主）+ 陈晓峰（辅）
- **人工智能类** → 李思源博士（主）+ 张明远教授（辅）
- **商业科技类** → 王建华教授（主）+ 周文博（辅）
- **游戏开发类** → 陈晓峰（主）+ 周文博（辅）
