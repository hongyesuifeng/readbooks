# CLAUDE.md

此文件为 Claude Code (claude.ai/code) 在此代码仓库中工作时提供指导。

## 笔记质量规范（最高优先级）

每次生成读书笔记或游戏分析时，必须遵守 `.claude/rules.md` 中定义的批判性审查标准。

**核心要求：**
- 笔记完成初稿后，必须用 `rules.md` 第一节的自我审查问题逐项检查
- 笔记深度目标：至少 3 个第四层分析（产生洞察），1 个第五层洞察（改变认知）
- 借鉴点必须落地到具体项目/文件/功能，不接受泛泛的"可以参考这个思路"
- 发现 command/skill 有结构性问题时，当场更新，不拖延

详见 `.claude/rules.md`

## 项目概述

这是"ReadBooks"项目 - 一个个人知识沉淀平台。用于系统性地记录读书笔记、游戏分析中的思考、总结和知识体系构建。

## 项目目标

1. **读书笔记管理** - 为每本书创建详细的读书笔记
2. **游戏分析记录** - 对玩过的游戏进行系统拆解，沉淀设计认知
3. **知识架构生成** - 使用 Mermaid 等工具绘制知识体系图
4. **深度衍生思考** - 针对内容进行延伸思考和多角度分析
5. **知识关联网络** - 建立书籍/游戏间的关联关系，形成知识图谱
6. **后续规划** - 基于当前内容推荐后续阅读/游玩方向
7. **学习进度跟踪** - 记录阅读/游玩状态和个人成长轨迹

## 工作流程

当用户要求为某本书生成读书笔记时：

1. **信息收集** - 使用 WebSearch 搜索书籍相关信息
2. **模板选择** - 根据书籍类型选择合适的模板
3. **专家分析** - 自动选择相关专家并生成深度分析
4. **知识关联** - 分析与已读书籍/游戏的关联关系
5. **后续推荐** - 基于书籍内容推荐后续阅读路径
6. **笔记生成** - 创建符合模板的完整读书笔记
7. **质量检查** - 使用质量标准检查清单验证笔记质量
8. **文件组织** - 将笔记保存到 `books/` 对应分类子目录下
9. **知识图谱更新** - 更新 `books/知识关联图谱.md`

当用户要求为某款游戏生成分析笔记时：

**纯策划视角（/game-notes）：**
1. **信息收集** - 使用 WebSearch 搜索游戏信息、评测、设计分析
2. **分类判断** - 确定游戏类型，放入 `games/` 对应子目录
3. **生成分析** - 按游戏分析模板生成完整笔记，重点关注系统架构和设计取舍
4. **知识关联** - 与已读书籍的理论建立关联
5. **图谱更新** - 更新 `games/游戏知识图谱.md`

**全维度分析（/game-full-analysis）：**
1. **信息收集** - 同上，多平台搜索 + WebFetch 提取详情
2. **源码分析**（有 sourcePath 时）- 目录结构总览 → 技术栈识别 → 核心模块定位 → 关键系统代码阅读
3. **分类判断** - 确定游戏类型，放入 `games/` 对应子目录
4. **生成 Part 1** - 策划视角深度分析（比 game-notes 更深，含约束条件分析）
5. **生成 Part 2**（有源码时）- 技术架构与实现分析（选型分析、模块架构、核心系统实现、性能工程实践、技术债）
6. **生成 Part 3**（有源码时）- 策划 ↔ 程序打通分析（设计意图→代码映射、技术约束→设计影响、文档vs现实）
7. **生成 Part 4** - 综合总结 + 知识关联
8. **质量检查** - 按 `.claude/rules.md` 策划/技术/打通三项标准逐项审查
9. **图谱更新** - 更新 `games/游戏知识图谱.md`

详细流程见 `.claude/workflows/读书笔记标准流程.md` 和 `.claude/skills/game-full-analysis.md`

## 读书笔记标准格式

每本书的笔记必须包含：

### 基础结构（必需）
- 📚 基础信息（书名、作者、出版社等）
- 📖 内容概要（书籍简介、核心主题、主要章节）
- 🧠 知识架构（Mermaid 思维导图）
- ✍️ 读书笔记（重点摘录、个人思考、实践应用）
- 📊 学习总结（收获、观念改变、未来行动）

### 深度分析（⭐⭐⭐⭐以上）
- 💭 深度衍生思考（核心观点延伸、多角度分析、创新思考）
- 🎓 专家视角分析（多专家深度分析和观点整合）
- 🔗 知识关联网络（与已读书籍的关联、概念映射）
- 📚 后续阅读路径规划（直接延伸、交叉验证、实践补充）

### 质量标准
- 所有笔记应达到⭐⭐⭐⭐以上质量等级
- 优秀笔记应达到⭐⭐⭐⭐⭐质量等级
- 质量标准详见 `templates/笔记质量标准.md`

### 技术书籍专项规范（强制遵守）

**技术书籍**（编程、游戏开发、软件工程、算法等）的笔记必须遵循以下规范：

1. **每章独立展开**：不能将多个章节合并成一节，每章必须单独深度展开，不能用一两句话带过。

2. **核心概念和原理为主体**：每个知识点先讲"是什么、为什么这样设计、解决了什么问题"，而不是直接跳到"怎么写代码"。

3. **架构图和设计图为主要表达方式**：
   - 优先使用 Mermaid 流程图、时序图、架构图说明系统设计和模块关系
   - 优先使用对比表格说明不同设计方案的取舍
   - 优先使用 ASCII 框图展示层次结构、数据流和执行顺序
   - 能用图说清楚的，不用代码和大段文字

4. **代码仅作为辅助理解的内容**：
   - 代码出现前必须先有概念说明
   - 代码篇幅不超过章节总内容的 30%
   - 只有当概念本身必须用代码才能完整表达时才写代码
   - 禁止用代码替代概念解释

5. **对比分析**：每个重要概念都应与相关领域的其他方案对比（例如 Godot 节点模型 vs Unity 组件模型），揭示设计取舍背后的原因。

## 常用命令

### 读书笔记生成
- **创建新书笔记**: `读书笔记 {书名}` - 自动搜索并生成完整笔记
- **专家分析**: `专家分析 {书名}` - 生成多专家视角深度分析
- **快速笔记**: `读书笔记 {书名} simple` - 生成简化版笔记

### 游戏分析生成
- **创建游戏笔记**: `/game-notes {游戏名}` - 纯策划视角，自动搜索并生成完整游戏分析
- **全维度分析**: `/game-full-analysis {游戏名}` - 策划+程序双视角深度分析
  - 提供源码时：策划 + 技术架构 + 策划-程序打通，三板块完整输出
  - 无源码时：自动降级为深度加强版策划分析
  - 用法：`/game-full-analysis 背包乱斗 sourcePath=D:/projects/backpack`

### 知识管理
- **查看书籍图谱**: 查看 `books/知识关联图谱.md`
- **查看游戏图谱**: 查看 `games/游戏知识图谱.md`
- **质量检查**: 使用 `templates/笔记质量标准.md` 检查笔记质量
- **更新关联**: 完成后更新对应知识图谱

### 工作流程
- **标准流程**: 参考 `.claude/workflows/读书笔记标准流程.md`
- **模板选择**: 根据内容类型选择合适的模板
- **专家选择**: 根据书籍主题自动选择相关专家

## 注意事项

1. **始终使用中文** - 所有对话和笔记内容都使用中文
2. **模板化生成** - 使用标准化模板确保笔记格式一致
3. **个性化内容** - 在标准格式基础上添加个人思考和感悟
4. **知识可视化** - 重视思维导图和架构图的绘制
5. **版权尊重** - 摘录内容要注明出处，避免大段引用

- 更新书籍和游戏知识图谱
- 提供阅读/游玩建议和扩展学习资源
- 协助整理和归纳学习成果
- 确保笔记质量达到⭐⭐⭐⭐以上等级

## 专家Agent系统

项目内置了多位资深专家Agent，可从不同专业视角分析书籍内容，帮助提炼核心观点。

### 可用专家

| 专家 | 领域 | 专长 |
|------|------|------|
| 张明远教授 | 计算机科学 | 系统架构、算法、编程语言理论 |
| 李思源博士 | 人工智能 | 机器学习、深度学习、AI应用 |
| 王建华教授 | 商业科技 | 科技战略、商业模式、投资分析 |
| 陈晓峰 | 游戏客户端 | 游戏引擎、图形渲染、性能优化 |
| 周文博 | 游戏行业 | 游戏设计、市场分析、产业发展 |

详细定义见 `agents/experts.md` 和 `agents/system.md`

### 使用方式

1. **单专家分析**:
   ```
   请用张明远教授的视角分析《编译原理》
   ```

2. **多专家分析**:
   ```
   请从计算机科学和人工智能角度分析这本书
   ```

3. **专家圆桌讨论**:
   ```
   请组织专家圆桌讨论这本书的核心价值
   ```
